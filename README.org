#+TITLE: org-resolutions

* Overview

~org-resolutions.el~ evaluates Org checkbox items from named numeric rules.
Rules can return a numeric threshold result or a ~(GOAL CURRENT)~ pair; the
latter renders each child item as a percentage cookie like ~[27%]~.

Example Org list:

#+begin_src org
- [33%] #+resolutions
  - [27%] drink-water 4 5 4
  - [48%] bike-miles 13 16
  - [100%] read-pages 70
#+end_src

* Add It To Your .emacs

If the files live in =~/src/org-resolutions/=, add that directory to your
~load-path~, load the library, define your rules, and enable the mode.

#+begin_src emacs-lisp
(add-to-list 'load-path "~/src/org-resolutions")

(require 'org-resolutions)

(setq org-resolutions-rules
      '(("drink-water" . (lambda (args) (list 48 (apply #'+ args))))
        ("bike-miles" . (lambda (args) (list 60 (apply #'+ args))))
        ("read-pages" . (lambda (args) (list 60 (apply #'+ args))))))

(org-resolutions-mode 1)
#+end_src

With that in place, pressing ~C-c C-c~ on the ~#+resolutions~ marker item or on
one of its children will evaluate the list and refresh the checkbox cookie.

* Use The Example Configuration

This repository also includes ~example-usage.el~, which contains a fuller sample
rule set. To use that from your =~/.emacs=:

#+begin_src emacs-lisp
(add-to-list 'load-path "~/src/org-resolutions")

(require 'org-resolutions)
(load "example-usage")
#+end_src

* Org Syntax

Each child item under the marker should have this form:

#+begin_src text
NAME ARG1 ARG2 ...
#+end_src

The ~NAME~ is looked up in ~org-resolutions-rules~, and the numeric arguments
are passed to the corresponding function as a list.

If a rule returns ~(GOAL CURRENT)~, the child item is shown as
~[floor(100 * CURRENT / GOAL)%]~, clamped to 100%.  Parent percentage cookies
still behave like normal Org checkbox lists: only children at 100% count as
complete, so the parent is based on completed children rather than averaging
the child percentages.

That means a line like ~read-pages 70~ with a goal of ~60~ must display
~[100%]~, not ~[0%]~, because values above the goal are capped rather than
wrapping or resetting.

In the example above, only one of the three children is complete, so the
parent must display ~[33%]~.
